Functions

  • Named Functions

    • Named functions are declared with the function keyword and then the function name then the parameters of the function. Named functions are not executed immediately but are executed when the function is called on. An example of a named function is

    function functionName(a) {
    return a;
    }

    Here you can see that the function is declared with the function keyword. Next we named the function functionName. The parameters of the function is a and it is returned in the body of the function.

  • Function Expression

    • Another example of a function is a function expression. These functions are stored in a variable. Then later in your code you can use the variable as a function.

    var z = function (a) {
    return a;
    }

    Here you can see that the function is stored into the variable z. We still need the function keyword but then we don't need a name for the function because the variable is the name. It also has a parameter a and it is returned in the body of the function.

  • Arrow Functions

    • An arrow function is the shortest way to write a function as you don't need the function keyword, return, and the curly brackets that both of the other functions use. Arrow functions are not hoisted so you will have to define them before use. The function could be as simple as this.

    const z = (a) => a;

    The variable z gets assigned the function with parameters a and the function will return a. Another way to write this as an arrow function is
    const x = (a) => { return a };

What else can you do with functions?

  • Optional Arguments

    • Optional arguments allow named parameters to be initialized with default values. An example of this would be

    function functionName(a, b = 10) {
    return a * 10;
    }
    console.log(functionName(5))
    console.log(functionName(5, 5))

    In the first console.log you will multiply 5 times 10 because there was not a second parameter. In the second console.log you will get 25 because the second parameter 10 gets overridden by 5.

  • Rest Parameters

    • Rest parameters allow any number of parameters as an array.

    function functionName(a, ...moreParams) {
    console.log(a)
    console.log(moreParams)
    }
    functionName("one", "two", "three", "four")

    console.log(a) will return one while console.log(moreParams) will return the rest of the parameters that have been enters so two, three, four.

  • Scope

    • There are two types of scope in javascript global scope and local scope. If a variable is declared outside of a function is it global and any functions can access it. If a variable is declared inside of a variable it will be in the functions scope and code outside of the function will not be able to use that variable.

  • Nested Functions

    • A nested function is just what it sounds like a function inside of another function. The scope of these nested functions is that the functions have access to the variables above them.

    function addItself(a) {
    var num = a;
    function plus() {
    num += a;
    }
    plus();
    console.log(num);
    }
    addItself(5);

    This function will set num to 5 and then go into the plus function and add another 5 to num. The function plus can still access the a parameter even though it has no parameters itself. The output of this function will be the number the function takes times 2.
