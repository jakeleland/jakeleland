APIs

  • What are APIs?

    • APIs stands for the Application Programming Interface. It allows two applications to talk to each other. An API is used to send data from a messenger to the system then delivers the response back to the messenger. This helps us receiver data without having to input it all into a database. There are some very good APIs that make coding rather simple but there can also be very bad APIs that are hard to get the data from.

  • REST

    • REST stands for Representational state transfer and is a software architectural style. RESTful web services provides communication between the computer systems on the internet.

  • Postman

    • Postman is used as an API client that can be used to create, share, test and document APIs.

  • API Code

    • This is my API code that I've been working on for my future project in this class. The API code sends a GET request to the host "api-nba-v1.p.rapidapi.com" and I also send my API key to that. The url specifics what API I want to hit and this specific request hits teams and then I used the Chicago Bulls team ID of 6. The ajax request will return the team name as an object.

    var teamName = {
    "async": true,
    "crossDomain": true,
    "url": "https://api-nba-v1.p.rapidapi.com/teams/teamId/6",
    "method": "GET",
    "headers": {
    "x-rapidapi-host": "api-nba-v1.p.rapidapi.com",
    "x-rapidapi-key": "MY API CODE"
    }
    };

    $.ajax(teamName).done(function (response) {
    var team = response['api']['teams'][0];
    console.log(team);
    document.getElementById('full-name').innerHTML = team['fullName'];
    document.getElementById('logo').src = team['logo'];
    document.getElementById('short-name').innerHTML = team['shortName'];
    });

    The return of this will be Chicago Bulls, the Bulls logo, and CHI.
